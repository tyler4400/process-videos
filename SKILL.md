# 把视频教程转成跟敲文档

你的工作是：基于**预处理好的字幕 + 截图**，读懂教学视频的代码变更，产出一份用户能跟着敲的 Markdown 文档。

> 配套工具：https://github.com/（发布后填 url）—— `preprocess-videos.sh` 负责把视频预处理成 `video-notes-cache/`。
> 不要自己去下载模型或做耗时转写，**先用预处理产物**。

---

## 何时使用本 Skill

- 用户提供 `.mp4` / `.mkv` / `.mov` 等视频路径
- 用户说"整理视频"、"跟着视频敲代码"、"把视频转成文档"、"视频做了哪些代码变更"
- 用户要把多节视频合并成一份总文档

## 何时**不**使用

- 用户只想随口问视频讲什么 → 直接用语音/字幕摘要即可，不必走完整流程
- 视频不是编程教学（比如生活类、演讲）→ 可以借鉴字幕提取步骤，但文档结构不适用

---

## 工作流程（必读）

### 第 0 步：定位缓存

**最重要的优化点**：字幕和截图可能**已经存在**于视频同级的 `video-notes-cache/`，**千万不要重新转写**（单个 10 分钟视频用 medium 模型要 15-30 分钟）。

给定视频路径 `DIR/VIDEO_NAME.mp4`，缓存位于：

```
DIR/
├── VIDEO_NAME.mp4
└── video-notes-cache/
    └── VIDEO_NAME/
        ├── transcript.srt      ← 带时间戳的字幕
        ├── transcript.txt      ← 纯文本字幕
        ├── frames/             ← 每 10s 一张截图（默认值，可配置）
        │   ├── frame_001.jpg
        │   └── ...
        └── .fingerprint        ← 缓存指纹
```

检查步骤：

1. 给定视频 `DIR/VIDEO_NAME.mp4`，直接 `Read` 这两个路径看是否存在：
   - `DIR/video-notes-cache/VIDEO_NAME/transcript.txt`
   - `DIR/video-notes-cache/VIDEO_NAME/transcript.srt`
2. 如果找到 → 直接进第 1 步
3. 如果没找到 → 告诉用户先运行预处理脚本。**脚本同时接受目录或单视频路径**：

```bash
# 只处理这一个视频（推荐：最快，且后续再对整个目录跑时会自动跳过这一个）
~/Tools/process-videos/preprocess-videos.sh "<视频文件绝对路径>"

# 或批量处理整个目录
~/Tools/process-videos/preprocess-videos.sh "<视频所在目录>"
```

**不要**自己调 ffmpeg / whisper 手动转写，这会浪费几十分钟且用户体验差。

### 第 1 步：理解视频内容

按**效率优先顺序**阅读：

| 资源 | 读取方式 | 作用 |
|---|---|---|
| `transcript.txt` | `Read` 工具 | 先整体扫一遍讲了什么（最快） |
| `transcript.srt` | `Read` 工具 | 需要定位时间戳时再读 |
| `frames/frame_XXX.jpg` | `Read` 工具传图片路径 | **交叉验证代码画面**，非常关键 |

**关键原则：字幕告诉你"说了什么"，截图告诉你"写了什么"**。光读字幕会误判（讲师会说"这个函数我们等会儿写"，究竟是当场写了还是留到下一节很容易搞错）。务必看几张关键帧对照。

### 第 2 步：交叉验证项目代码

打开用户项目里相关的源文件，对比"视频里的最终代码"与"项目里的当前代码"：

- 如果一致 → 按视频讲的写文档
- 如果不一致（项目里更新） → 提示用户"项目里是 XX 版，视频里是 YY 版"，通常按**视频里的版本**写（因为是跟敲教程）

### 第 3 步：产出文档

按下方【文档模板】写。文档放在用户项目的 `docs/video-notes/` 下（如果这个目录不存在，创建一个）。

## 文档模板

### 模板 A：单视频跟敲文档

适用于"帮我整理这一节视频"这类请求。

```markdown
# <视频标题> —— 跟敲笔记

> 视频：《<章节>·<标题>》
> 时长：约 X 分 Y 秒
> 笔记依据：whisper 语音转写 + 关键帧截图 + 项目代码上下文

---

## 一、本节需求 & 目标
<讲师开头说的本节要做什么，从 transcript.txt 的前几十行提取>

## 二、本节要改的 N 个文件
<一张表，列出每个文件和改动概述>

## 三、分步跟敲
### 步骤 1：<名称> `⏱ MM:SS - MM:SS`
<具体代码 + 为什么这么写>

### 步骤 2：<名称> `⏱ MM:SS - MM:SS`
...

## 四、跟敲完成后的验证
<预期现象、控制台输出、对照表等>

## 五、本节核心代码汇总（对照参考）
<完整/关键片段>

## 六、下节预告
<讲师末尾提到的下一节做什么>

## 七、常见跟敲踩坑提示
<常见错误和排查办法>
```

### 模板 B：合集文档（多视频合并）

适用于"把这 N 节视频合并成一份，按功能重组，一步到位"这类请求。

```markdown
# <章节名> · <主题> 合集（XX-a 到 XX-b）—— 一步到位跟敲笔记

> 合并自 N 节视频：
> - XX-a <标题>
> - XX-b <标题>
> ...
>
> 本文按**功能模块**重组，不按视频顺序，避免"先写错再改"的返工。

---

## 一、整体需求 & 目标
<合并后的最终效果、功能清单、数据流设计图>

## 二、本章要改 / 新建的 N 个文件
<一张表>

## 三、分步跟敲
<按【从底层到表层】或【按文件分组】组织，而不是按视频顺序>
<每一步不需要时间戳>
<注意：合并后的步骤应该是最终状态，避免"先这样再改"的返工>

## 四、完整代码参考（对照检查用）
<按文件分节给出增量>

## 五、关键知识点回顾
<在本章里用到的核心技术/概念>

## 六、课后练习 / 优化
<讲师留的作业 + 可选优化方向>

## 七、常见跟敲踩坑
<汇总所有视频里的踩坑>
```

## 时间戳规范（仅模板 A 使用）

- 格式：`⏱ MM:SS - MM:SS`（没有波浪线）
- 放在每个"步骤 N：XXX"标题的**右侧**
- 精度到秒，从 SRT 字幕里读
- 每个步骤对应字幕里讲解该步骤的时间段

**如何从 SRT 提取时间**：SRT 格式是

```
17
00:01:23,720 --> 00:01:27,820
最后保存的时候还没有可能有一些项目信息
```

取 `00:01:23` 作为该内容的开始时间。定位每个步骤的起止时间需要读上下文。

## 关键踩坑（我之前犯过的错）

### 1. 讲师说"我们这里已经建好了" ≠ 不用写

讲师常常说"我这里已经建好了 XX 组件"——这不是"已经写完所有代码"，而是"文件已经创建好但内容还要现场敲"。一定要看**截图里的光标位置**判断到底写到哪一行。

### 2. 跨视频的"返工陷阱"

某个视频里讲师写了一个空壳函数，留着"下节课再补"。如果你在给**前一节**写文档时照抄了**完整最终版**，就会骗用户"你这节就该写完整"。正确做法：

- 对比前后两节的截图时间线
- 在文档里明确标注"此处 setup 的事件处理函数留到下节 XX-XX 补齐"

### 3. 贪婪读取字幕导致上下文丢失

不要只读前 50 行字幕就开始写文档，至少**整份通读一遍**再动笔。讲师经常在视频中段做回顾或修正。

### 4. 截图密度

预处理默认每 10 秒一张。10 分钟视频约 60 张。不要每张都读，挑：
- 每 5-10 张读一张做整体理解
- 关键代码编辑时刻**必读**
- 界面效果演示时刻**必读**

### 5. 项目代码可能是"最终版"

视频是历史版本，项目里可能经过多次迭代。写文档时遵循**视频里的版本**，在开头或踩坑区注明"项目当前版本是 XX，比视频更进一步"。

## 产出物清单

完成后你应该交付：

- ✅ 一份或多份 Markdown 文档（放 `docs/video-notes/`）
- ✅ 文档无 lint 错误（用 `ReadLints` 工具验证）
- ❌ **不要**主动搬运字幕到视频目录（除非用户要求）
- ❌ **不要**自己下载 whisper 模型或运行耗时任务

---

## 常用工具调用速查

```
Read("<DIR>/video-notes-cache/<VIDEO_NAME>/transcript.txt")
  → 先读纯文本版快速扫一遍（<VIDEO_NAME> 是去扩展名的视频文件名）

Read("<DIR>/video-notes-cache/<VIDEO_NAME>/transcript.srt")
  → 需要时间戳时读 SRT

Read("<DIR>/video-notes-cache/<VIDEO_NAME>/frames/frame_015.jpg")
  → 读截图（支持 jpg/png）

Glob("video-notes-cache/**/transcript.srt", target_directory=<视频所在目录>)
  → 批量发现：当用户让你整理一整个章节的多个视频时用

ReadLints(["<doc path>"])
  → 最后检查文档
```
